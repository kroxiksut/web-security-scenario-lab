# CASE-TEMPLATE.md Guide

Сами кейсы должны вестись на английском языке. Для нового кейса используйте шаблон из `CASE-TEMPLATE.md` без перевода полей.

## Как использовать шаблон

- скопируйте `CASE-TEMPLATE.md` в нужную категорию
- переименуйте файл по правилам идентификаторов
- заполните front matter
- заполните разделы кейса на английском языке
- при необходимости redacted-данные оставляйте redacted, а не восстанавливайте опасные значения

## Обязательные поля

- `caseId`
- `module`
- `category`
- `status`
- `sourceType`
- `firstSeen`
- `reproducibleInLab`
- `expectedSignal`
- `shouldFire`
- `severity`
- `tags`
- `coverageDimensions`

## Пояснения

- `publicUrl`: реальный URL только если безопасно публиковать
- `safeDisplayUrl`: безопасный вид адреса, например `example[.]com`
- `scenarioMapping`: путь или идентификатор сценария в lab, если он уже есть
- `notes`: краткие служебные заметки в metadata

## Когда кейс можно считать хорошим

Хороший кейс:

- безопасен для публикации
- понятен reviewer'у без запуска живого сайта
- содержит detector-relevant signals
- даёт основу для будущего сценария или уже маппится на текущий
