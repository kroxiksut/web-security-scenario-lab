# Кейсы категории Prompt Splitting

## Область

Кейсы с фрагментированной доставкой prompt, распределением инструкций по разным DOM-областям, отложенной сборкой или многочастными паттернами prompt assembly.

## Когда использовать эту категорию

Используйте эту категорию, если доминирующее семейство сигналов относится к `prompt-splitting`.

Если кейс одинаково затрагивает несколько модулей, лучше перенести его в `cross-module/`.

## Именование

Рекомендуемый префикс файлов: `PS-YYYY-NNN.md`

## Минимальные ожидания

- файл кейса ведётся на английском языке
- основа берётся из `../CASE-TEMPLATE.md`
- указано ожидаемое поведение детектора
- заполнены coverage dimensions
- указано, воспроизводится ли кейс в lab

## Что здесь может появляться дальше

- сами кейсы
- локальные скриншоты или ссылки на материалы в `../assets/`
- маппинг на сценарии стенда
