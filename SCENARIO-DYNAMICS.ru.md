# Динамика сценариев

## Цель

Стенд не должен опираться только на один фиксированный HTML-образец для каждого кейса. Каждое семейство сценариев должно поддерживать контролируемую вариативность, чтобы поведение детекторов можно было проверять на наборе близких, но отличающихся состояний страницы, а evaluation outputs при этом оставались воспроизводимыми.

## Режимы динамики

- `static`: фиксированная и воспроизводимая разметка
- `seeded`: случайная, но воспроизводимая генерация по seed
- `live-reroll`: новый вариант при обновлении страницы или ручном reroll
- `timed-dynamic`: содержимое меняется после загрузки страницы

## Источники мутаций

- замена текста
- замена стилей
- перестановка CSS-классов
- изменение фонов
- вставка скрытых узлов
- отложенное появление overlay
- перегенерация ссылок
- замена доменных меток
- изменение атрибутов

## Воспроизводимость

Случайность должна контролироваться через seed.

- одинаковый seed -> одинаковый результат
- разный seed -> разные варианты
- одинаковый seed -> одинаковая evaluation metadata для одной версии сценария
- seed должен быть виден в UI
- seed должен приниматься через query string

## Перегенерация ссылок

Для `link-domain-security` движок должен поддерживать:

- ротацию несовпадений между видимым текстом и `href`
- ротацию гомографических кандидатов
- примеры с punycode и mixed-script
- подстановку redirect-параметров
- замену unsafe protocols
- замену target hostname

## Визуальная перегенерация

Для `visual-manipulation` движок должен поддерживать:

- скрытый текст с разными техниками сокрытия
- случайные сочетания цвета текста и фона
- варианты clipping и off-screen позиционирования
- варианты opacity и filter
- изменение размеров overlay и `z-index`
- вставку скрытых input или editable-областей на разной глубине DOM

## Модели запуска изменений

Динамические обновления должны запускаться через:

- загрузку страницы
- refresh
- кнопку reroll
- таймерную мутацию
- имитацию пользовательского действия

## Связка с evaluation metadata

Мутации сценария должны оставаться связаны с evaluation metadata.

- каждый сгенерированный вариант должен приводиться к expected signal profile
- подозрительные и benign-варианты должны различаться в metadata
- coverage tags должны сохраняться при reroll внутри одного семейства сценариев
- объяснения в `why flagged` panel должны строиться на активном варианте, а не только на статическом тексте

## Правила безопасности

- Никакого динамического исполнения кода из недоверенных данных.
- Никакой сборки HTML во время runtime из внешних источников.
- Шаблоны должны приходить только из локальных манифестов или локального кода.
- Опасный контент только имитируется, а не принимается от пользователя.
