# Дорожная карта

## Этап 1: Определение

- утвердить название проекта и его объём
- зафиксировать static-only архитектуру
- определить базовую матрицу совместимости браузеров
- определить семейства сценариев для первых двух модулей
- определить placeholder-структуру для:
  - `trigger-phrases`
  - `prompt-splitting`
  - `api-interception`
- определить базовые evaluation fields и coverage dimensions

## Этап 2: MVP стенда

- создать landing page
- сделать навигацию по модулям
- реализовать словари RU и EN
- реализовать движок seed-управляемых сценариев
- реализовать static и dynamic сценарии для:
  - `visual-manipulation`
  - `link-domain-security`
- реализовать документированные страницы-заглушки для:
  - `trigger-phrases`
  - `prompt-splitting`
  - `api-interception`
- реализовать базовые evaluation outputs для рабочих сценариев
- реализовать первую версию `why flagged` panel

## Этап 3: Повышение устойчивости

- добавить страницы с `jQuery`, `React` и `Vue`
- добавить покрытие `Shadow DOM` и `iframe`
- добавить проверки по браузерной матрице
- добавить smoke-тесты на Playwright
- опубликовать правила для внешних вкладов в planned modules
- добавить первые представления coverage matrix

## Этап 4: Масштабирование

- добавить манифесты для следующих модулей и классов сценариев
- развить заглушки в полноценные наборы сценариев для:
  - `trigger-phrases`
  - `prompt-splitting`
  - `api-interception`
- расширить структуру каталога сценариев
- расширить таксономию evaluation metadata
- добавить новые языки
- документировать browser-specific ограничения

## Этап 5: Поддержка адаптации под другие браузеры

- использовать тот же стенд при адаптации браузерных расширений или других ИБ-инструментов под дополнительные браузеры
- сохранять стенд независимым от API расширения
- использовать стенд как регрессионный набор для browser-specific сборок
- развить benchmark-lite слой дальше простого показа страниц
