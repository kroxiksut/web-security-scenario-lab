# Дорожная карта

Этапы поставки — внутренний словарь планирования, он живёт только в этом файле: документация
организована по темам, а не по этапам. Текущее состояние: этапы 1–3 завершены, этап 4 в основном
выполнен, этап 5 не начинался.

## Этап 1: Определение — завершён

- утвердить название проекта и его объём
- зафиксировать static-only архитектуру
- определить базовую матрицу совместимости браузеров
- определить семейства сценариев для первых двух модулей
- определить placeholder-структуру для:
  - `trigger-phrases`
  - `prompt-splitting`
  - `api-interception`
- определить базовые evaluation fields и coverage dimensions

## Этап 2: MVP стенда — завершён

- создать landing page
- сделать навигацию по модулям
- реализовать словари RU и EN
- реализовать движок seed-управляемых сценариев
- реализовать static и dynamic сценарии для:
  - `visual-manipulation`
  - `link-domain-security`
- реализовать документированные страницы-заглушки для:
  - `trigger-phrases`
  - `prompt-splitting`
  - `api-interception`
- реализовать базовые evaluation outputs для рабочих сценариев
- реализовать первую версию `why flagged` panel

## Этап 3: Повышение устойчивости — завершён

- добавить страницы с `jQuery`, `React` и `Vue`
- добавить покрытие `Shadow DOM` и `iframe`
- добавить проверки по браузерной матрице
- внедрить движок сценариев на TypeScript + тулчейн Vitest/Ajv/ESLint/Prettier (тесты только
  на движок/инфраструктуру; страницы сценариев намеренно не покрываются тестами)
- добавить опциональный/ручной E2E на Playwright против расширения PageCheck (не гейтящий CI)
- опубликовать правила для внешних вкладов в planned modules
- добавить первые представления coverage matrix

## Этап 4: Масштабирование — в основном выполнен

Сделано:

- заглушки развиты в полноценные наборы сценариев для `trigger-phrases`, `prompt-splitting` и
  `api-interception`, у каждого есть benign-контроль ложных срабатываний
- в схему манифеста введён контракт benign-сценариев (`whyFlagged` / `whyBenign`)
- добавлен каталог сценариев, строящийся из данных, и обязательное поле `page` в каждом манифесте
- расширена таксономия evaluation metadata (теги, coverage dimensions, пометки ahead-of-detector)
- манифесты заполнены полем `detectionExamples`

Осталось:

- отрисовывать `detectionExamples` рядом со сценарием в UI (данные есть, представления нет)
- пробелы покрытия из раздела *Планируемое покрытие* в документе каждого модуля (benign-контроль
  теперь есть у каждого модуля)
- добавить новые языки
- документировать browser-specific ограничения
