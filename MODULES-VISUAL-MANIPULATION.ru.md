# Покрытие модуля Visual Manipulation

## Цель

Подготовить страницы сценариев, которые нагружают эвристики скрытого контента и CSS-манипуляций на реалистичных DOM-структурах.

## Обязательные семейства сценариев

- скрытый текст
- скрытые input
- скрытые editable-поверхности
- overlay
- CSS-обфускация
- смешанные техники сокрытия

## Статические сценарии

- текст скрыт через `display: none`
- текст скрыт через `visibility: hidden`
- текст скрыт через нулевую opacity
- текст вынесен за пределы экрана
- текст скрыт через clipping
- очень мелкий шрифт и слабый контраст
- полноэкранный overlay с фиксированным позиционированием

## Динамические сценарии

- скрытые узлы добавляются после загрузки
- текст переключается между видимым и скрытым состоянием
- overlay появляется с задержкой
- стили переписываются по таймеру
- контраст фона и текста перегенерируется
- скрытый input добавляется рядом с видимыми формами

## Варианты сложности DOM

- плоский DOM
- вложенные контейнеры
- компоненты, отрисованные библиотечными страницами
- контент внутри shadow root
- образцы внутри iframe, где это уместно

## Требования к страницам

- главная страница модуля со списком сценариев
- метаданные по каждому сценарию
- поддержка reroll
- поддержка seed
- видимое пояснение, какие техники сокрытия сейчас активны

## Критерии успеха

- авторы детекторов быстро прогоняют несколько вариантов сокрытия
- сценарии воспроизводимы по seed
- динамические изменения выявляют проблемы, чувствительные к таймингу
- стенд не требует активной блокировки или DOM-вмешательства
