# Добавление фреймворка или версии (Фаза 3)

Английское зеркало: [`../en/adding-frameworks.md`](../en/adding-frameworks.md).

Фреймворк-страницы воспроизводят сценарий через конкретный UI-рантайм (jQuery, React, Vue, …),
чтобы детектор PageCheck проверялся на DOM, созданном фреймворком, а не только на рукописной
разметке. **Версия библиотеки — измерение покрытия:** каждый мажор даёт свою сигнатуру DOM/рантайма,
поэтому один и тот же сценарий размещается на нескольких закреплённых версиях.

## Жёсткие правила

- **Бандлить локально — никакого runtime CDN.** Библиотека — зависимость времени сборки; рантайму
  сеть не нужна.
- **Точные пины, без `^`/`~`.** Устаревшие/легаси-версии выбраны намеренно и не должны
  автообновляться.
- **Один мажор фреймворка на страницу.** Никогда не грузить два мажора одной библиотеки на одной
  странице.
- **Сценарные страницы не покрываются unit-тестами.** Тестируется только инфраструктура
  движка/матрицы (см. `AGENTS.md`). Сама матрица (`data/frameworks.json`) валидируется.
- **Никаких реальных брендов или живых доменов** в контенте сценариев (только зарезервированные
  имена RFC 2606).

## Выбор стратегии установки

| Форма библиотеки | Стратегия | Пример |
| --- | --- | --- |
| Самодостаточная (нет межпакетных импортов) | **Root npm-алиас** | jQuery, Vue: `jquery1@npm:jquery@1.12.4`, `vue2@npm:vue@2.7.16` |
| В рантайме зависит от соседнего пакета | **Вложенный `node_modules`** в `frameworks/<lib>/<vN>/` + Vite-алиас | React: `react-dom` внутри импортирует голый `react` |

Случай React ломается с обычным root-алиасом, потому что внутренний `import "react"` у `react-dom`
привяжется к неправильной версии. Вложенная установка кладёт каждый `react-dom` рядом с его парным
`react`, а Vite `resolve.alias` указывает import-специфер на этот вложенный путь; поскольку
алиасный `react-dom` физически лежит рядом со своим `react`, его собственный голый импорт
резолвится (по расположению файла) в co-located версию.

## Добавить новую ВЕРСИЮ существующего фреймворка

1. **Установить пин.**
   - Самодостаточная: `npm install --save-exact "<alias>@npm:<pkg>@<version>"`.
   - Вложенная (React): добавить `frameworks/<lib>/<vN>/package.json` с точными `dependencies`,
     добавить папку в скрипт `frameworks:install`, выполнить `npm run frameworks:install` и добавить
     запись `resolve.alias` в `vite.config.ts` на вложенный `node_modules`. Вложенные `node_modules`
     в gitignore; сам `package.json`-пин коммитится.
2. **Типы для алиасов** (только если у специфера нет своих типов): добавить `declare module`-реэкспорт
   в `src/frameworks.d.ts`. Алиасам на пакет со своими типами (например `vue2`) ничего не нужно.
3. **Поведение** — добавить `src/scenarios/<id>.ts` с экспортом `run(ctx)`. Импортировать фреймворк
   через его алиас/специфер и делегировать общему драйверу в `src/scenarios/_shared/`. Переиспользовать
   существующий драйвер, если модель рендера совпадает; добавить ветку mount/адаптер, если API
   отличается (например React 17 `ReactDOM.render` vs 18/19 `createRoot`; Vue 2 `new Vue().$mount`
   vs 3 `createApp`).
4. **Страница** — добавить `frameworks/<lib>/<folder>/hidden-text.html`. Скопировать страницу
   существующей версии; задать `data-scenario="<id>"`, `data-module` и `data-root="../../.."`.
5. **Манифест** — добавить `data/scenarios/<id>.json` (та же ground truth; пометить фреймворк и
   версию, например `framework:react`, `legacy-version`).
6. **Vite-вход** — зарегистрировать страницу в `rollupOptions.input` в `vite.config.ts`.
7. **Матрица** — добавить запись версии под фреймворком в `data/frameworks.json`:
   `{version, major, folder, alias, releaseDate, whyIncluded (en+ru), scenarios: [{id, page}]}`.
8. **Обнаруживаемость** — coverage-view (`pages/frameworks/index.html`) покажет её автоматически из
   матрицы. Опционально добавить карточку на странице каталога модуля.

## Добавить НОВЫЙ фреймворк

То же, что выше, плюс:

- Добавить новую запись верхнего уровня в `data/frameworks.json` (`id`, `name`, `kind`, `versions`).
- Добавить новый общий драйвер в `src/scenarios/_shared/`, если ни одна существующая модель рендера
  не подходит.
- Задать нужные флаги сборки в `vite.config.ts` (например defines Vue `__VUE_OPTIONS_API__` при
  использовании без `@vitejs/plugin-vue`).

## Проверка (всё должно пройти)

```
npm run typecheck && npm run lint && npm run test && npm run build
```

Затем убедиться в выводе сборки, что фреймворк попадает в **свой отдельный код-сплит-чанк** и **не**
появляется в общем чанке `main`, и открыть страницу (`npm run dev`), чтобы увидеть рендер сценария.
Для фреймворков с вложенной установкой на свежем клоне сначала выполнить `npm run frameworks:install`.
