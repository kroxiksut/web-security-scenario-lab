# Визуальные манипуляции

Английское зеркало: [`../en/visual-manipulation.md`](../en/visual-manipulation.md).

Размещает на странице текст, который человек не видит, но DOM по-прежнему содержит: CSS-подавление,
границы DOM (shadow root, дочерние фреймы) и рендеринг, которым управляет фреймворк, — чтобы детектор
можно было проверить на сокрытии, переживающем любой распространённый способ сборки страницы.

Это самый крупный модуль стенда: **22 сценария из 36**, потому что одна и та же нагрузка
воспроизводится в 18 комбинациях «фреймворк + версия».

## Что он детектирует

Контент может присутствовать в DOM, быть доступным всему, что читает страницу программно, — и при
этом оставаться невидимым для человека перед экраном. Именно этот зазор и есть модель угрозы:
инструкции, маркеры или обманный текст проносят мимо человека, сохраняя их полностью доступными для
всего, что разбирает документ.

Задача модуля `visual-manipulation` — распознавать подавленный, но присутствующий контент независимо
от:

- **Техники** — почти нулевая непрозрачность, вынос за пределы экрана, клиппинг, субпиксельный размер
  шрифта или оверлей, перекрывающий контент по `z-index`.
- **Механизма стилизации** — инлайновые стили, записанные скриптом, классы из таблицы стилей или
  утилитарный CSS (`sr-only` в Tailwind), где подавление видно только через *вычисленный* стиль.
- **Границы DOM** — light DOM, открытый shadow root, вложенный shadow root на границу глубже или
  same-origin дочерний фрейм, созданный через `srcdoc`.
- **Рантайма рендеринга** — рукописная разметка либо DOM, созданный и управляемый React, Vue, Svelte,
  Angular, Solid, Preact, Lit, jQuery, htmx или Alpine, включая намеренно устаревшие мажорные версии.
- **Тайминга** — в каждом сценарии есть кнопка, добавляющая новые скрытые узлы, иногда со случайной
  задержкой: это нагружает обработку мутаций в детекторе, а не разовое сканирование страницы.

## Сценарии

21 позитивный сценарий разделяет один ground truth: `expectedSignal: hidden-content-manipulation`,
`shouldFire: true`, `severity: high`. Различаются они осью модели угрозы, которую нагружают, — она и
записана в `tags` и `coverageDimensions`. Один benign-контроль утверждает противоположный вердикт.

### Основные страницы

| id | страница | нагружаемая ось |
| --- | --- | --- |
| `hidden-text-mixed` | `pages/visual-manipulation/hidden-text.html` | База: opacity, вынос за экран, клиппинг и маскировка через z-index на одной рукописной странице |
| `hidden-text-shadow-dom` | `pages/visual-manipulation/shadow-dom.html` | Открытый shadow root плюс вложенный внутренний shadow root — рекурсивный обход |
| `hidden-text-iframe` | `pages/visual-manipulation/iframe.html` | Same-origin дочерний фрейм через `srcdoc` — спуск в `contentDocument` |
| `hidden-text-benign` | `pages/visual-manipulation/benign.html` | **Benign-контроль** (`shouldFire: false`): те же техники, применённые в легитимном UI |

### `hidden-text-benign` — контроль ложных срабатываний

- **Страница:** `pages/visual-manipulation/benign.html`
- **`shouldFire`:** `false` · **severity:** `low` · **expectedSignal:** `none`
- **Измерения покрытия:** текст только для скринридера (техника клиппинга), skip-ссылка, появляющаяся
  по фокусу, свёрнутый disclosure-виджет (`display: none`), неактивные ARIA-панели вкладок
  (`[hidden]`), вынесенные за экран слайды карусели, достижимость скрытого контента.
- **Почему безопасно:** каждый узел, невидимый при загрузке, невидим по обычной UI-причине, и
  пользователь может добраться до всего штатным взаимодействием — `sr-only`-подпись, описывающая
  кнопку-иконку, skip-ссылка, появляющаяся по фокусу, свёрнутое тело `<details>`, две неактивные
  панели ARIA-вкладок и запаркованные слайды карусели с отзывами. **Техники подавления намеренно те
  же, что в позитивных сценариях**: `.vm-sr-only` использует тот же приём `clip: rect(0 0 0 0)`, что
  и `.vm-clipped`, а карусель паркует слайды на `left: -9999px` ровно как позитивный сценарий с
  выносом за экран. Отличается содержимое (обычный товарный текст, а не пронесённые инструкции) и его
  достижимость. Детектор, реагирующий только на «текст есть, но не отрисован», сработает здесь и
  ошибётся.

Это страж ложных срабатываний модуля, часть общей benign-матрицы.

Каждая фреймворк-страница добавляет к общей нагрузке своё измерение, специфичное для рантайма: Lit
прячет контент внутри **shadow root**, отрисованного из шаблона, htmx вставляет его через
**out-of-band AJAX-swap**, Tailwind подавляет **утилитарными классами**, так что в инлайновом
атрибуте `style` не видно ничего, Angular работает в **zoneless-режиме с AOT-компиляцией**, Solid и
Svelte меняют DOM вообще **без virtual DOM**. Живая страница покрытия `pages/frameworks/index.html`
рисует эту матрицу из [`data/frameworks.json`](../../data/frameworks.json); как добавить библиотеку
или версию — см. [Добавление фреймворка или версии](./adding-frameworks.md).

## Как это детектируется

Техники сокрытия общие для всех сценариев и применяются к узлам, несущим реальный текст:

| Техника | Что делает сценарий | Почему человек этого не видит |
| --- | --- | --- |
| Непрозрачность | `opacity: 0.02` | Отрисовано, размечено, фактически невидимо |
| Вынос за экран | `position: absolute; left: -9999px` | За пределами вьюпорта, до него не доскроллить |
| Клиппинг | `position: absolute; clip: rect(0, 0, 0, 0)` | Обрезано до прямоугольника нулевой площади |
| Мелкий шрифт | `font-size: 0.1px` | Субпиксельный размер, нечитаемо |
| Маскировка z-index | Баннер-оверлей поверх контента | Перекрыто другим элементом |

Принципиально, что инлайновыми стилями это пишет только первая страница. Сценарий Tailwind добивается
того же результата утилитарными классами (`sr-only`), поэтому детектор, читающий `element.style`, не
увидит ничего — надёжный источник здесь `getComputedStyle`:

```js
for (const el of document.querySelectorAll("*")) {
  if (!el.textContent?.trim()) continue;
  const cs = getComputedStyle(el);
  const offScreen = parseInt(cs.left, 10) < -1000 || parseInt(cs.top, 10) < -1000;
  if (parseFloat(cs.opacity) < 0.05 || parseFloat(cs.fontSize) < 1 || offScreen) flag(el);
}
```

Одного `document.querySelectorAll("*")` не хватит для двух основных сценариев: контент в shadow root
и в дочернем фрейме невидим для запроса по light DOM и требует явного спуска.

```js
function* walk(root) {
  for (const el of root.querySelectorAll("*")) {
    yield el;
    if (el.shadowRoot) yield* walk(el.shadowRoot);              // открытые shadow root, рекурсивно
    if (el.tagName === "IFRAME" && el.contentDocument) yield* walk(el.contentDocument); // same-origin фреймы
  }
}
```

Поскольку инжектор в каждом сценарии может добавлять узлы **после** случайной задержки, единственное
сканирование на `DOMContentLoaded` по замыслу пропустит часть нагрузки; модуль и построен так, чтобы
нагружать `MutationObserver` (или эквивалентное повторное сканирование).

Проверка вычисленного стиля выше необходима, но **недостаточна**: примените её к `hidden-text-benign`
— и она пометит пять узлов, каждый из которых легитимен. Чтобы пройти оба сценария, нужен второй
вопрос: достижим ли этот контент и выполняет ли он роль в UI? На практике это означает не трогать
узлы, на которые ссылаются как на подписи доступности (`aria-describedby`, `aria-labelledby`), а
также те, что лежат внутри `[role="tabpanel"]`, закрытого `<details>` или карусели, либо становятся
видимыми по фокусу:

```js
const EXCUSED = '[role="tabpanel"], details, [aria-roledescription="carousel"], .skip-link';
function isLegitimatelyHidden(el) {
  if (el.id && document.querySelector(`[aria-describedby~="${el.id}"], [aria-labelledby~="${el.id}"]`))
    return true;
  return Boolean(el.closest(EXCUSED));
}
```

Все скрываемые строки берутся из небольшого набора явно помеченных лабораторных сниппетов
(`lab-marker: …`, `hidden note (test): …`), выбираемых по сиду, — конкретный текст варьируется
воспроизводимо и остаётся очевидно синтетическим.

## Планируемое покрытие

В исходных требованиях к покрытию модуля перечислено несколько семейств, которые **пока не поставлены
как сценарии**. Они остаются целью для расширения модуля, и ни одно из них не требует менять
информационную архитектуру:

- Статическое подавление через `display: none` и `visibility: hidden` как самостоятельные сценарии
  (сейчас затрагивается лишь косвенно).
- Скрытые и редактируемые поверхности форм — скрытый `input` рядом с видимыми элементами формы,
  скрытые области `contenteditable`.
- Полноэкранный оверлей с фиксированным позиционированием и оверлей, появляющийся только с задержкой.
- Переписывание стилей по таймеру; переключение текста между видимым и скрытым состоянием.
- Перегенерация контраста фона и текста по сиду (слабый контраст как отдельная ось).
- Второй benign-контроль на оси фреймворков — легитимное сокрытие, созданное компонентным рантаймом,
  где сгенерированная фреймворком разметка усложняет проверку достижимости.

## Ground truth и оценка

Манифест каждого сценария в `data/scenarios/hidden-text-*.json` — источник истины о том, что детектор
обязан сделать на этой странице: `expectedSignal` называет семейство находки, `shouldFire` говорит,
требуется ли находка, `severity` фиксирует ожидаемый приоритет. `tags` и `coverageDimensions`
описывают, какие оси (техника, механизм стилизации, граница DOM, фреймворк, версия) нагружает
конкретная страница, — благодаря этому pass/fail детектора привязывается к конкретному измерению, а
не к странице целиком. Текст `whyFlagged` написан для человека-ревьюера; поле `detectionExamples`
носит иллюстративный характер — это неисполняемый справочный сниппет, а не код детектора.

## Замечания по безопасности

- Все скрываемые строки — синтетические лабораторные маркеры. Ничто из этого не является рабочей
  нагрузкой и не переносится копипастом в атаку.
- Сценарии не загружают внешний контент: дочерний фрейм собирается из инлайнового `srcdoc`, а
  единственный сценарий, который вообще делает запрос, — htmx-swap — читает статический фрагмент
  (`public/lab-fragments/htmx-hidden.html`) с собственного origin стенда.
- Все библиотеки фреймворков вендорятся и пинятся локально — никакой runtime-зависимости от CDN,
  поэтому страницы работают офлайн, а версии не «уплывают».
