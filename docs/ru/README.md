# Фаза 4 — модули AI / prompt-injection

Английское зеркало: [`../en/README.md`](../en/README.md).

Фаза 4 добавляет три сценарных модуля, проверяющих обнаружение PageCheck манипуляций на странице,
адресованных AI-агенту, — контент, написанный не для человека-читателя, а для AI-ассистента,
читающего страницу (инструкция в стиле prompt-injection, та же инструкция, разбитая по
DOM-фрагментам, чтобы обойти проверку одной строки, и API-подобные поверхности взаимодействия, к
которым агента могут подтолкнуть). Каждый модуль поставляет и **позитивные** сценарии (детектор
обязан сработать), и **benign-контроль ложных срабатываний** (детектор обязан промолчать), а
манифест каждого сценария в `data/scenarios/` — источник истины, по которому оценивается прогон
детектора.

## Модули

- [Триггер-фразы](./trigger-phrases.md) — синтетические инструкции в стиле prompt-injection,
  размещённые в видимом тексте и текстовых атрибутах (`title`, `aria-label`, `alt`), на английском
  и русском, включая вариант с обходом через нормализацию Unicode и benign-контроль на контекст
  цитаты/кода.
- [Дробление промпта](./prompt-splitting.md) — тот же корпус инструкций, раздробленный по
  последовательным DOM-узлам внутри одного локального региона (пробельная «spaced» и разрывающая
  слово «compact» сборка) либо по текстовым атрибутам цепочки элементов («attribute-chain»), каждый
  со своим benign-контролем (разброс по регионам / целые фразы и цепочка с обычной подсказкой).
- [Перехват API](./api-interception.md) — DOM-атрибуты (`action`, `src`, `data-api`,
  `data-endpoint`), несущие API-подобные маркеры (`api`, `/v1/`, `/graphql`), плюс benign-контроль
  без маркеров в значениях, а также вторая ось — впереди текущего сигнала детектора, — сравнивающая
  заявленный MIME-тип `data:`-URI с фактической сигнатурой первых байт, тоже со своим
  benign-контролем.

Остальную документацию лаборатории см. в [основном указателе документации](./index.md).
