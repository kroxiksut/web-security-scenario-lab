# Документация

English version of this file: [`README.md`](./README.md).

Документация лежит здесь и разделена по языкам:

- `docs/ru/` — документы на русском
- `docs/en/` — английское зеркало (те же имена файлов, переведённое содержимое)

Начинайте с **[`docs/ru/index.md`](./ru/index.md)** (или **[`docs/en/index.md`](./en/index.md)**) —
это карта документации; чтобы сразу поднять стенд, переходите к разделу
[Запуск](./ru/getting-started.md).

## Конвенции

- **Документация складывается в `docs/en/` и `docs/ru/` парами-зеркалами** (одинаковое имя файла в
  обоих каталогах). Обновляйте обе версии одним изменением.
- `README.md` и `LICENSE` остаются в корне репозитория (так принято для GitHub-лендинга); корневая
  пара `README.md` / `README.ru.md` следует принятому в корне суффиксному соглашению
  `FOO.md` / `FOO.ru.md`.
- Документы организованы **по темам** — модули детекции, модель сценариев, платформа и политика.
  Внутренние фазы поставки — это словарь планирования, ему место в [`ROADMAP.ru.md`](../ROADMAP.ru.md),
  а не в заголовках документов.
- Часть унаследованных документов пока лежит в корне репозитория (`ARCHITECTURE.md`,
  `EVALUATION-OUTPUTS.md`, `SCENARIO-DYNAMICS.md`, `BROWSER-COMPATIBILITY.md`, `I18N.md`, …). Они
  могут переезжать в `docs/` постепенно; до переезда источником истины остаются файлы в корне, и при
  правке любого из них нужно синхронно обновлять его пару `FOO.md` / `FOO.ru.md`.
