# Documentation

Documentation lives here, split by language:

- `docs/en/` — English documents
- `docs/ru/` — Russian mirror (same filenames, translated content)

## Conventions

- **New documentation goes in `docs/en/` and `docs/ru/` as mirrored pairs** (same filename in
  both). Update both in the same change.
- `README.md` and `LICENSE` stay in the repository root (GitHub landing conventions).
- Some legacy documents still live in the repository root (`ARCHITECTURE.md`,
  `EVALUATION-OUTPUTS.md`, `SCENARIO-DYNAMICS.md`, `ROADMAP.md`, `MODULES-*.md`, etc.). They may
  be migrated into `docs/` incrementally; until then they remain the source of truth in root.
- Start from `docs/en/index.md` (or `docs/ru/index.md`) for the documentation map.
