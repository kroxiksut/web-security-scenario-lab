# Internationalization

## Initial Scope

The first release must support:

- Russian (`ru`)
- English (`en`)

The architecture should allow additional languages later without changing the scenario engine.

## Rules

- UI strings are stored in locale dictionaries.
- Scenario metadata supports localized titles and descriptions.
- Shared labels must not be duplicated across pages.
- The selected language should be accepted through query string and persisted locally when appropriate.

## Suggested Structure

```text
src/i18n/
|-- index.js
|-- locales/
|   |-- en.json
|   `-- ru.json
```

## Localization Coverage

- landing page labels
- module names
- scenario names
- buttons and controls
- warnings and notes
- compatibility hints
- metadata panel labels

## Fallback Policy

- default locale: `en`
- if a translation is missing, fall back to `en`
- do not break page rendering on missing localized keys

## Future Expansion

When new locales are added:

- keep keys stable
- avoid mixing locale logic with scenario rendering logic
- allow localized scenario descriptions without changing scenario identifiers
